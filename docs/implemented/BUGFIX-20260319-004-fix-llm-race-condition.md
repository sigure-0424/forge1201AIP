# BUGFIX-20260319-004-fix-llm-race-condition

## Issue
The bot became trapped in an infinite loop where the system feedback from successful actions (e.g., `Successfully followed Player`) was unconditionally sent back to the LLM as a new input. This automatically triggered the LLM to generate the next action based on outdated context without waiting for the user. Additionally, there was no concurrency control; new user commands (like "stop") would dispatch alongside active actions, failing to interrupt background routines generated by the system loop.

## Resolution
1. **Concurrency Control**: Added `this.activeLlmRequests` tracking in `AgentManager.js` using timestamps (`thoughtId`) to ensure only one LLM request is valid at a time. Old async fetches are discarded if a newer thought supersedes them.
2. **User Command Interrupts**: High-priority user commands immediately send a `{"action": "stop"}` to flush the Actuator's `actionQueue`, clearing out background bot tasks and forcing it to accept the latest directive.
3. **State Management for System Feedback**: System messages beginning with `"Successfully"` now cause the `AgentManager` to transition into an *Idle* state instead of automatically querying the LLM for a new directive. Only failures (e.g., `"Failed"`, `"Cannot"`) will trigger the LLM for error recovery without user intervention, and only if another thought is not already active.
